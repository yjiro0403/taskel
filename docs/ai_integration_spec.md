# AI機能統合 仕様書

## 1. 概要

TaskelにAIアシスタント機能を搭載する。  
ユーザーが自然言語で指示することで、タスクの作成・提案・分析を行う。

### ユースケース

| # | 機能 | 入力例 | 出力 |
| :--- | :--- | :--- | :--- |
| UC-1 | タスク作成 | 「明日15時に歯医者」 | タスク自動生成（日付・タイトル・時刻を推定） |
| UC-2 | 一括タスク作成 | 「買い物リスト：牛乳、卵、パン」 | 3件のタスクを生成 |
| UC-3 | 今日の提案 | 「今日やること提案して」 | 未完了タスク・ルーティンから優先順位を提案 |
| UC-4 | タスク分析 | 「今週の振り返り」 | 完了率・時間消費の分析レポート |

## 2. 技術選定

### 2.1 LLM: Gemini 1.5 Flash（有料枠）

| 項目 | 内容 |
| :--- | :--- |
| **モデル** | `gemini-1.5-flash` |
| **利用枠** | Pay-as-you-go（有料枠） |
| **プライバシー** | ◎ 有料枠はモデル学習にデータ不使用 |
| **コスト** | Input: $0.075/1Mトークン, Output: $0.30/1Mトークン |
| **精度** | タスク抽出・提案に◎（Function Calling対応） |

### 2.2 SDK: Vercel AI SDK (`ai`)

Next.js App Routerと最も親和性が高いSDK。  
`useChat` フックでストリーミング対応のチャットUIを即座に構築可能。

### 2.3 アーキテクチャ

```
┌─────────────────┐     ┌──────────────────────────┐     ┌─────────────────┐
│   Frontend      │     │   Next.js API Route      │     │   External API  │
│                 │     │   (Server-side)           │     │                 │
│  AIChatPanel    │────▶│  /api/ai/chat/route.ts   │────▶│  Gemini API     │
│  (useChat hook) │◀────│                          │◀────│                 │
│                 │     │  Tools (Function Calling) │     │                 │
│                 │     │   ├─ createTask           │     │                 │
│                 │     │   ├─ getTodayTasks        │     │                 │
│                 │     │   └─ suggestPlan          │     │                 │
│                 │     │                          │     │                 │
│                 │     │  /api/tasks/route.ts     │     │  Firestore      │
│                 │     │  (既存BFF)               │────▶│  (DB)           │
└─────────────────┘     └──────────────────────────┘     └─────────────────┘
```

**ポイント**:
- AIの処理はすべてサーバーサイド（API Route）で実行
- APIキーはサーバーの環境変数に格納し、クライアントには露出しない
- タスク作成はFunction Callingでスキーマを厳密に定義し、既存BFFを内部で再利用

## 3. プライバシー方針

### 送信データの最小化

AIに送信するデータは以下に限定する：
- タスクの `title`, `date`, `status`, `estimatedMinutes`, `actualMinutes`
- セクションの `name`, `startTime`
- ルーティンの `title`, `frequency`

**送信しないデータ**:
- `userId`, `id` 等の内部ID
- `memo`（個人的なメモ）
- `attachments`（添付ファイル）
- プロジェクト招待や認証情報

### プライバシーポリシー更新

AI機能のリリース時に `docs/PrivacyPolicy.md` を更新し、以下を明記：
- AI機能はGoogle Gemini APIを使用
- 送信されるデータの範囲
- データはモデル学習に使用されない旨

## 4. コスト方針

### フェーズ1（初期リリース）
- 開発者側でAPIキーを負担（月額推定: 数十円〜数百円）
- 1ユーザーあたりの利用回数は制限しない（自然な利用量なら問題なし）

### フェーズ2（ユーザー拡大時）
- BYOK（Bring Your Own Key）機能を設定画面に追加
- または有料プランにAI機能をバンドル

## 5. キー判断事項

| 判断 | 決定 | 根拠 |
| :--- | :--- | :--- |
| LLMモデル | Gemini 1.5 Flash | コスト最安・精度十分・Function Calling対応 |
| 利用枠 | 有料枠 (Pay-as-you-go) | 無料枠はデータが学習に使われるリスクあり |
| SDK | Vercel AI SDK | Next.js App Router と最も親和性が高い |
| UI | FABボタン + スライドパネル | 既存UIを壊さず追加可能 |
| データ送信 | 最小限（タイトル・日付のみ） | プライバシー保護 |
