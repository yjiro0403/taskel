## 検証対象

Daily Review サマリ生成機能の実装を検証する。

### 実装ファイル
- `src/lib/ai/types.ts` - `DailyReviewData`型定義
- `src/lib/ai/tools.ts` - `getDailyReview`ツール実装（L516-L715）
- `src/components/ai/DailyReviewSummary.tsx` - UIコンポーネント
- `src/components/ai/ChatMessage.tsx` - ルーティング（L156-L164）
- `src/lib/ai/prompts.ts` - プロンプト更新（L44, L62-L66）

### 機能概要
ユーザーがAIチャットで「今日の振り返り」等のリクエストをすると、以下の情報をサマリカードとして表示する:
- 完了タスク一覧（見積もり vs 実績時間）
- 未完了タスク一覧（open / in_progress）
- スキップされたタスク
- 全体統計（完了率、見積もり精度）
- 目標別進捗（Goal紐づけタスク）
- AIからのフィードバックメッセージ

## 10. バグ報告

### [POTENTIAL-001] Goal削除後のorphan参照
- **重大度**: Low
- **影響範囲**: `getDailyReview`ツール
- **状況**: タスクに`parentGoalId`が設定されているが、Goalが削除済みの場合、goalTitlesマップに存在せず、goalProgressに含まれない
- **期待動作**: 問題なし（削除済みGoalは表示不要）
- **実際の動作**: 仕様通り
- **評価**: ✅ **問題なし** - 意図的な挙動

### [POTENTIAL-002] Firestoreインデックス未作成の可能性
- **重大度**: Medium
- **影響範囲**: `getDailyReview`ツール
- **状況**: `userId`+`date`の複合クエリを実行しているが、Firestoreインデックスが未作成の可能性
- **期待動作**: クエリが成功する
- **実際の動作**: インデックスがない場合、Firestoreがエラーを返す（エラーハンドリングで空のサマリにフォールバック）
- **修正案**: Firebase Consoleまたは`firestore.indexes.json`で以下のインデックスを作成
```json
{
  "indexes": [
    {
      "collectionGroup": "tasks",
      "queryScope": "COLLECTION",
      "fields": [
        { "fieldPath": "userId", "order": "ASCENDING" },
        { "fieldPath": "date", "order": "ASCENDING" }
      ]
    }
  ]
}
```
- **評価**: ⚠️ **本番デプロイ前に要確認**
